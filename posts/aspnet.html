ASP .NET Core

Gestão de API REST
Voltar

Conceito:

ASP .NET Core é framework web, que usa C# em plataforma .NET


Estrutura:

  • PastaProjeto/
    • Controllers/: recebe request, retorna response. Não contém regras de negócio
    • DTOs/: Data Transfer Objects, controla saída e entrada da API. Evita expôr Model direto
    • Services/: regra de negócio, contém lógica de aplicação
    • Repositories/: acesso a dados, isolando banco de dados
    • Models/: entidades com propriedades e validações
    • Data/: EF Core, configuração do contexto do banco de dados
    • Program.cs: ponto de entrada da aplicação, configura host e serviços

Funcionamento:

  1. Cliente envia request para endpoint da API
  2. Controller recebe request, valida dados e chama Service
  3. Service executa lógica de negócio, chama Repository para acessar dados
  4. Repository interage com banco de dados, retorna resultado para Service (response)
  5. Service processa resultado, retorna resposta para Controller
  6. Controller formata resposta e envia de volta ao cliente

API REST básica:

  1. Instalar SDK .NET, VSCode, e extensão 'C# Dev Kit'
  2. Criar projeto: 
    
dotnet new webapi -n NomeDoProjeto
cd NomeDoProjeto
code .
dotnet add package Microsoft.EntityFrameworkCore.InMemory
dotnet add package Swashbuckle.AspNetCore

dotnet run
- Acessar http://localhost:5173/swagger para testar endpoints
