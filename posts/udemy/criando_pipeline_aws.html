Conceito:

Curso teórico e prático para criação de Pipeline CI/CD na AWS free tier, para automatizar implantação de API REST backend Django. Etapas via console AWS, e acessos via AWS CloudShell e máquina terceira.


Objetivo:

O objetivo do curso é implantar (deploy) API REST backend desenvolvida com tecnologia Python Django DRF, através da plataforma AWS FreeTier, utilizando serviço CodePipeline, que tem como função criar Pipeline CI/CD para automatizar fluxo da implantação de projetos de código. Após implantação do projeto via Pipeline, cujo repositório source encontra-se no GitHub, API REST backend Django será acessada e consumida via AWS CloudShell e máquina terceira Ubuntu Server WSL.


Pré-requisitos:

O curso será desenvolvido via ambiente Windows, utilizando recursos nativos do sistema operacional, como WSL. Porém, pode-se também utilizar outros ambientes desktop, dadas devidas especificidades de cada sistema operacional. As etapas desenvolvidas no console AWS serão as mesmas para todos sistemas operacionais.

  • Conhecimentos de Git, GitHub, programação web básica e comandos básicos Linux;
  • Conta GitHub;
  • Conta no Postman;
  • Conta AWS FreeTier (necessário cartão de crédito, sem custos dentro do limite gratuito);

Pipeline:

Pipeline é passo-a-passo automatizado para implantar projeto de software. Inclui etapas desde captura do código (source), execução (build), testes (test) e implantação (deploy).


CI/CD:

CI (Continuous Integration - Integração Contínua) é integrar frequentemente código em repositório central, executando builds e testes automáticos para detectar erros rapidamente. CD (Continuous Delivery/Deployment - Entrega/Implantação Contínua) é entrega automatizada do software, via Pipelines, garantindo que ele esteja sempre disponível em ambiente de produção. Resumidamente, CI integra e valida código automaticamente em local, onde CD pegará esse código e o implantará automaticamente em produção, via Pipelines.


API REST:

API REST é projeto web back-end com suporte para comunicação entre sistemas externos, utilizando verbos HTTP REST (GET - visualizar, POST - adicionar, PUT - atualizar e DELETE - deletar), contemplando suas respectivas funcionalidades de CRUD (create, update, retrieve e delete). Consumir API REST é executar as funcionalidades suportadas deste projeto web REST backend. Consumo de API REST geralmente é feito por outro projeto frontend, paralelo a esse, com outras tecnologias que suportam consumir sistemas API REST backend, via verbos HTTP REST. A API REST utilizada nesse curso é projeto sobre books com funcionalidades de CRUD:

  • Create book (POST - http://localhost:8000/books/)
  • Edit book (PUT - http://localhost:8000/books/{id}/)
  • View book (GET - http://localhost:8000/books/{id}/)
  • List books (GET - http://localhost:8000/books/)
  • Delete book (DELETE - http://localhost:8000/books/{id}/)

Cada book possui title (string), author (string) e description (string).


Django DRF:

Django REST Framework (DRF) é tecnologia que utiliza linguagem de programação base Python para desenvolvimento web, suportando padrão de utilização de verbos HTTP REST, sendo ideal para criar APIs web REST backend.
